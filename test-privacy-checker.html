隐私保护机制

隐私保护承诺：您的邮箱已在本地加密，我们不会传输和存储原始信息。只发送哈希值前缀到API进行查询。

保护流程

  1. 本地加密：使用SHA-1算法在浏览器中加密您的邮箱地址
  2. 提取前缀：只取哈希值的前5个字符作为查询前缀
  3. API查询：将前缀发送到Have I Been Pwned API
  4. 本地匹配：在浏览器中匹配完整的哈希值
  5. 结果展示：显示查询结果，不存储任何信息

技术保障

  • 使用crypto-js库进行本地加密
  • 符合Have I Been Pwned API设计规范
  • 只传输哈希值前缀，不泄露完整信息
  • 所有处理都在用户浏览器中完成
  • 不存储任何用户数据

使用指南

  1. 输入邮箱：在邮箱输入框中输入您要检查的邮箱地址
  2. 开始查询：点击"检查泄露"按钮开始查询过程
  3. 等待结果：系统会显示"检查中..."状态，通常需要几秒钟
  4. 查看结果：查询完成后会显示邮箱安全状态
  5. 安全建议：根据查询结果查看相应的安全建议
  6. 采取行动：如果发现泄露，立即采取安全措施

API查询流程

1. 邮箱输入：用户在界面中输入邮箱地址
2. 本地加密：使用SHA-1算法加密邮箱，生成40位哈希值
3. 提取前缀：取哈希值的前5个字符作为查询前缀
4. API调用：发送GET请求到 https://api.pwnedpasswords.com/range/{prefix}
5. 获取结果：API返回包含该前缀的所有哈希值后缀列表
6. 本地匹配：在浏览器中匹配完整的哈希值
7. 显示结果：根据匹配结果显示安全状态和泄露次数
// 核心查询代码示例 const hash = CryptoJS.SHA1(email.toLowerCase().trim()).toString().toUpperCase() const prefix = hash.substring(0, 5) const suffix = hash.substring(5) const response = await fetch(`https://api.pwnedpasswords.com/range/${prefix}`) const data = await response.text() // 在本地匹配完整哈希值 const lines = data.split('\n') for (const line of lines) { const [hashSuffix, count] = line.split(':') if (hashSuffix === suffix) { // 发现泄露 return { isSafe: false, breachCount: parseInt(count, 10) } } }

技术实现

核心技术栈

  • Vue 3 Composition API：现代化响应式开发框架
  • crypto-js：JavaScript加密库，提供SHA-1加密功能
  • Have I Been Pwned API：权威的数据泄露查询服务
  • Fetch API：现代浏览器网络请求接口
  • 响应式设计：支持桌面端和移动端
